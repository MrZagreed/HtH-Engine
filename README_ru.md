# Highway to Hell Engine (Пре-релиз)

[English version](README.md)

Синхронизация Spotify -> Discord Rich Presence в реальном времени с текстами песен.

## Быстрый старт без лишних знаний
1. Скачай/распакуй проект в любую папку.
2. Запусти `run.py` (двойной клик) или через терминал:
   - `py -3.12 run.py`
3. При первом запуске сначала выбери язык (`English` по умолчанию или `Русский`), затем режим работы.
4. Пройди шаги мастера настройки.
5. Готово: приложение само поставит зависимости и стартует.

## Что автоматизировано
- проверка совместимости Python (приоритет 3.12)
- создание/пересоздание venv при необходимости
- установка зависимостей из `requirements.txt`
- перезапуск внутри venv
- мастер первого запуска на EN/RU

## Где лежат данные проекта
Создаются автоматически при первом запуске:
- `highway_to_hell_engine_data/config/highway_to_hell_engine_config.json`
- `highway_to_hell_engine_data/cache/lyrics_cache.db`
- `highway_to_hell_engine_data/cache/spotify_oauth_cache.json`

Если в корне был старый конфиг, он мигрируется автоматически.

## Режимы
### LOCAL (без Premium)
- нужен только `Discord Client ID`
- читает реальную позицию трека из Windows Media Session
- при необходимости использует fallback по заголовку окна Spotify

### API (с Premium)
- нужны:
  - `Discord Client ID`
  - `Spotify Client ID`
  - `Spotify Client Secret`

## Где получить ключи
### Discord Client ID
1. Открой Discord Developer Portal: [discord.com/developers/applications](https://discord.com/developers/applications)
2. Создай/выбери приложение
3. Скопируй `Application ID` (это и есть `Discord Client ID`)

### Spotify API ключи
1. Открой Spotify Developer Dashboard: [developer.spotify.com/dashboard](https://developer.spotify.com/dashboard)
2. Создай приложение
3. Скопируй `Client ID` и `Client Secret`
4. В настройках приложения добавь redirect URI:
   - `https://127.0.0.1:8888/callback`

## Мини-туториал: LOCAL
1. Запусти Discord
2. Запусти Spotify и включи трек
3. Выполни `py -3.12 run.py --mode=local`
4. Введи Discord Client ID в мастере
5. Проверь видимость активности:
   - Discord Settings -> Activity Privacy -> `Display current activity as a status message`

## Мини-туториал: API
1. Запусти Discord
2. Выполни `py -3.12 run.py --mode=api`
3. Введи Discord + Spotify ключи в мастере
4. Заверши авторизацию Spotify через браузер

## Источники лирики (приоритет)
1. LRCLIB (synced/plain)
2. Genius
3. Musixmatch
4. AZLyrics
5. lyrics.ovh

Кэш хранится в SQLite (`lyrics_cache.db`).

## Поведение Rich Presence
- hover-текст картинки: `Discord Karaoke RPC by Mr.Zagreed`
- безопасная кнопка `Open in Spotify` (валидация URL)
- ограниченная частота RPC-обновлений
- псевдографический прогресс-бар в Rich Presence

## Диагностика обрывов сети
- отслеживаются состояния `online/degraded/offline`
- в логах пишутся длительность outage, количество сбоев и последняя ошибка
- в API режиме используется адаптивный backoff
- в LOCAL режиме трекинг продолжается с уменьшенной нагрузкой

Не учебное задание и не короткий эксперимент, а полноценный инструмент, который можно довести до законченного состояния.

Этот проект стал для меня первым по-настоящему серьёзным самостоятельным проектом. До этого у меня были небольшие программы и эксперименты, но именно здесь я впервые решил пройти весь путь разработки: от идеи и первых строк кода до стабильного и завершённого результата.

Изначально всё выглядело очень просто. Первая версия проекта представляла собой один монолитный файл примерно на 300 строк кода. Никакой архитектуры фактически не было. Всё писалось быстро и работало только потому, что код был небольшим.

Со временем проект начал меняться. По мере обучения в колледже я постоянно открывал для себя новые подходы и инструменты разработки. Многие знания, которые я получал на занятиях, постепенно переносились в этот проект. Я переписывал части системы, делил код на модули, улучшал структуру и старался сделать архитектуру более понятной и устойчивой.

Разработка шла не всегда ровно. Бывали периоды, когда из-за сложности задач или нехватки опыта я откладывал проект на время. Но спустя какое-то время возвращался к нему снова, уже с новыми знаниями и пониманием того, как всё должно работать.

Постепенно простой монолитный скрипт превратился в полноценный модульный проект. Этот процесс занял почти год и стал для меня важной практикой. Он научил меня терпению, системному мышлению и тому, как постепенно улучшать собственный код.

Сегодня, **10 марта 2026 года**, когда я заканчиваю четвёртый курс и мне уже 20 лет, я могу сказать, что наконец довёл проект до состояния, которым действительно доволен.
